## 데이터베이스의 특징
```
데이터베이스란? 
- 여러 사람이 공유하고 사용할 목적으로 통합 관리되는 정보의 집합이다.

- 그 내용을 고도로 구조화함으로써 검색과 갱신의 효율화를 꾀한 것이다

- 데이터베이스를 관리하는 시스템(DBMS)과 통합되어 제공되어야 함. (간단히 비유하면 데이터베이스는 데이터를 저장하는 창고, DBMS는 창고 속 데이터 처리자)

특징들
- 다수의 사용자에게 동시 접근을 허용한다(동시성 제어)

- 프로그램과 데이터의 격리 : 데이터베이스는 자기기술성을 가지므로 저장 구조 등을 수정하는 것이 응용 프로그램에 영향을 미치지 않는다.

- 동일한 내용의 데이터가 중복되어 있지 않아야 하고,  검색이나 갱신이 효율적으로 이루어질 수 있도록 해야 한다.

```
## 데이터베이스의 종류
이곳에 작성하지 않은 종류도 있지만, 채팅방 서비스에 적합하지 않다고 생각하여 포함시키지 않았습니다.
```
1. 관계형 데이터베이스
현재 가장 많이 사용되고 있는 데이터베이스의 한 종류,
관계형 데이터베이스는 테이블(table)로 이루어져 있으며, 이 테이블은 키(key)와 값(value)의 관계를 나타냅니다.
이처럼 데이터의 종속성을 관계(relationship)로 표현하는 것이 주된 특징

장점)
- 데이터의 분류, 정렬, 탐색 속도가 빠르다

- 오랫동안 사용된 만큼 신뢰성이 높고, 데이터의 무결성을 보장.

단점)
- 기존에 작성된 스키마를 수정하기가 어렵다

- 데이터가 2차원 표형태로만 출력되기 때문에 트리 구조로 조직화되는 '객체'들과 궁합이 잘 안맞는 게 문제

2. 계층형 데이터베이스
데이터의 관계를 트리 구조로 정의함. 상위의 레코드가 복수의 하위 레코드를 가지는 구조
데이터 간에 1:N으로 종속적 관계를 지니는 것이 주된 특징

장점)
- 1:N관계를 갖는 대용량 데이터처리에 강력
- 데이터 무결성이 좋은 편

단점)
- 데이터 중복의 문제가 생김
- 구조를 변경하기 어려운 편

3. 객체형 데이터베이스
말그대로 데이터베이스에 객체 지향 개념을 도입하여 만들어짐.
객체들을 생성하여 class에서 체계적으로 정리하고, 
하위 계층이 상위 계층으로부터 속성과 방법들을 물려받을 수 있는 계승 가능한 구조로 구성
즉 자신의 데이터 일부를 포함하는 유사한 class을 하위 class(Maintenance Activity)으로 만들어 둔 데이터베이스


장점)
사용자가 정의한 사용자 정의 타입을 지원한다.
객체들 사이의 참조 구조를 이용한 접근이 가능하다.
객체지향 프로그래밍 언어들과 객체지향 데이터베이스의 스키마 구조가 유사하다.

단점)
기본적인 데이터베이스 기능에 있어 안정성과 성능이 떨어진다.
```

## 설계 순서
```
데이터베이스 설계 단계는 요구 조건 분석, 개념적 설계, 논리적 설계, 물리적 설계, 구현 단계로 이루어져 있다.

-요구 조건 분석
사용자가 무엇의 정보를 필요로 하는지를 분석하여 어떤 데이터베이스를 만들어야 할지를 결정하는 단계

-개념적 설계
데이터베이스의 구성요소(속성)들을 정하고, 다음 단계인 논리적 설계에서 수행할 작업을 설계

-논리적 설계
본격적인 스키마 설계로, 이전 개념적 설계에서 만들어 놓은 설계도를 사람들이 알아볼 수 있는 형태로 작업하는 것. 제약조건을 이 단계에서 제작.

-물리적 설계
저장장치에 저장되는 형태의 알고리즘을 효율적이게 개선하고 수정하는 단계다

-구현
 실제로 자료들을 수집한 뒤 데이터베이스의 알맹이들을 채워 넣는 단계
```
